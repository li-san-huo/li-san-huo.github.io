  1. 选择器：也叫选择符，用于选择页面上要添加样式的标签。

    • 基础选择器：单个选择器

      • 全局选择器（通配符选择器）-选择页面上所有的标签

    • 标签选择器（元素选择器）：选择页面上的同种标签

    • ID选择器：选择页面上的唯一的一个元素

    • 类别选择器：选择页面上class命名相同的标签

    • 基础选择器的优先级

    • 复合选择器：由两个或者是多个基础选择器组合而成的选择器

      • 群组选择器（集体声明）：多个选择器设置同样的样式

    • 后代选择器：用空格连接，选择指定标签元素下的后辈元素

    • 子选择器：用>连接，选择指定元素的第一代子元素

css三种布局机制

  • 普通流（标准流）：按照默认方式排列

  • 浮动

    浮动可以让多个块级盒子一行没有缝隙排列显示，经常用于横向排列盒子。

  • 定位

    让盒子自由的在某个盒子内移动位置或者固定屏幕中某个位置，并且可以压住其他盒子。

定位属性的使用

边偏移

定位属性position

  • static(默认值) –静态定位
    • 对元素的排列没有影响，普通流显示
    • 几乎不用
  • relative–相对定位
    • 元素只有设置相对定位，没有偏移，对元素的排列没有任何影响
    • 偏移时参照元素原来的位置
    • 原始位置保留
    • 几乎不单独使用，一般给绝对定位作参照

  • absolute–绝对定位

    • 完全脱离文档流，在页面上不占空间

    • 父级没有定位，以浏览器（文档顶点）为参照进行偏移

    • 父级有定位，元素将以最近的已经定位（相对、绝对、固定）的父级（祖先）元素进行偏移。

      案例：子绝父相–父相（给子元素做参照）子绝（确定子元素的位置）

  • fixed–固定定位
    • 完全脱离文档流
    • 以文档的可视窗口为参照进行偏移
  • sticky–粘性定位（了解）
    • 以浏览器的可视窗口为参照点移动元素（固定定位特点）
    • 粘性定位占有原先的位置（相对定位特点）
    • 必须添加top、left、right、bottom其中一个才有效
    • 兼容性较差，IE不支持

层叠属性

  • 在使用定位布局时，可能会出现盒子重叠的情况。使用定位的盒子，默认后面的盒子会压住前面的盒子。

定位扩展

  • 绝对定位的盒子居中
  • 定位特殊性
  • 脱标的盒子不会触发外边距塌陷
  • 绝对定位（固定定位）会完全压住盒子
