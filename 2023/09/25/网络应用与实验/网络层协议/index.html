网络层协议

一、IP地址概述与应用

  1. IP地址概述

    • 定义：是主机的唯一表示，也是一种网络编码，保证计算机在网络中能够正常通信。
    • 特点：
      • 表示形式：点分十进制，例如：192.168.10.10
      • IP地址组成：32位的二进制组成，分为两部分：其一，网络部分（network）；其二，主机部分 （host）。
    • IP地址分类：
      • A类、B类、C类、D类、E类。但常用有三类地址：A类、B类和C类。
        1. A类地址：
          • 范围：1~126（0 ~ 127，其中排除0，一般代表子网网段，127,一般代表本机地址）
          • 组成：ClassA = 网络部分+主机部分+主机部分+主机部分
          • 应用场景：一般用于大型网络规划使用的IP地址
        2. B类地址：
          • 范围：128~191
          • 组成：ClassB = 网络部分+网络部分+主机部分+主机部分
          • 应用场景：一般用于中型网络规划使用的IP地址
        3. C类地址：
          • 范围：192~223
          • 组成：ClassC = 网络部分+网络部分+网络部分+主机部分
          • 应用场景：一般用于小型网络规划使用的IP地址
    • IP地址分类之二（企业应用的付费方式上划分）
      • 公网地址（公有地址），企业付费使用，一般用于企业外网的互联网。属于A/B/C类
      • 私网地址（私有地址），企业免费使用，一般用于企业内部的局域网。
        1. A类私网地址范围：10.0.0.0~10.255.255.255
        2. B类私网地址范围：172.16.0.0~172.31.255.255
        3. C类私网地址范围：192.168.0.0~192.168.255.255
          • C类私网地址可用主机IP地址的范围，例如：192.168.10.0
          • 总IP地址范围：192.168.10.0~192.168.10.255
          • 192.168.10.0代表网络地址，不能作为主机IP地址使用
          • 192.168.10.255代表广播地址，也不能作为主机IP地址使用
          • 可用主机IP地址范围：192.168.10.1~192.168.10.254

  2. 子网掩码

    • 子网掩码作用——用于确定IP地址的网络通讯（网络地址也称为“网段”）
    • 子网掩码：
      • 32个二进制位组成
      • IP地址的网络部分用1表示
      • IP地址的主机部分用0表示
    • 表示形式：点分十进制
    • 子网掩码分类
      • A类IP地址默认子网掩码：255.0.0.0
      • B类IP地址默认子网掩码：255.255.0.0
      • C类IP地址默认子网掩码：255.255.255.0

  3. IP地址应用

    • 配置网卡信息时，必须配置内容：IP地址和子网掩码。（网关地址和DNS服务器地址是可选项）
    • 查看windows主机网卡的地址信息
      • ipconfig (查看简要信息)
      • ipconfig /all（查看详细信息）
    • 将IP地址和子网掩码，同时转换为二进制，且做与运算，可得到该IP地址所处的网络地址，也称“网段”。
    • 网络测试命令
      • ipconfig（用于windows操作系统）
      • ifconfig （用于linux操作系统）
      • ping （测试网络或主机的联通性）
        • ping 127.0.0.1 测试本机主机是否正确的配置了TCP/IP信息。
        • ping 192.168.10.10 测试与本地主机是否能正常通信
        • ping 192.168.10.11 测试与其他主机是否能正常通信
        • ping 192.168.10.1 测试与默认网关地址是否能正常通信

  4. 路由器基本配置

    • 路由器初始化配置-1（环境准备）

    • 路由器初始化配置-2（环境准备）

    • 路由器的接口配置（路由器默认情况下，接口是关闭状态）

      • 命令格式：
        • Router(config-if)# ip address 192.168.10.1 255.255.255.0
        • Router(config-if)# no shutdown（激活接口）

    • 路由器的查询命令

      • 查看命令：
        • Router# show ip interface brief (查看路由器接口概要信息)
        • Router# show interface f0/1 (查看路由f0/1接口的详细信息)

二、IP数据包格式

  • TTL值——表示IP数据包在网络传输的生命周期，每经过一个路由器TTL值减1，若TTL=0，则路由器直接丢 弃该数据包。
  • TTL值——测试目标主机操作系统的类型
    • 通过ping命令测试，若TTL=128，代表目标主机为Windows操作系统；
    • 通过ping命令测试，若TTL=64，代表目标主机为Linux操作系统；
    • 通过ping命令测试，若TTL=255，代表目标为网络设备（路由器或三层交换机）；
  • 源IP地址——标识数据包来源主机IP地址（逻辑地址）
  • 目标地址——标识数据包目标主机IP地址（逻辑地址）

三、ICMP协议介绍

  • ICMP协议定义：是一种错误侦测与回馈机制
  • ICMP协议的对应的命令：基于windows操作系统的Ping命令用法如下：
    • ping -t 192.168.10.12 一直ping目标主机192.168.10.12
    • ping -n 6 192.168.10.12 指定目标主机192.168.10.12回应次数
    • ping -l 65500 192.168.10.12 指定目标主机每次ping包大小（字节）
    • ping -a 192.168.10.12 显示目标主机192.168.10.12的主机名称
  • 利用ICMP协议，发起攻击（Ping命令，发起泪滴攻击）
    • ping -a -t -l 65500 192.168.10.12

四、ARP协议介绍

  • 定义：arp：(地址解析协议)，将一个已知的IP地址解析为MAC地址
  • arp协议应用——通过arp广播获取目标主机IP地址对应的MAC地址，并将其保存到源主机的arp缓存表里。 主机arp缓存表记录内容的老化时间为120秒。

五、ARP攻击原理

  • arp攻击——导致网络无法正常通信，公司内部PC无法正常访问外网（Internet）。

  • arp欺骗：

    • 欺骗网关
    • 欺骗主机

  • arp欺骗——在不影响公司网络正常通信的前提，通过攻击主机获取公司网络内部信息，导致信息泄 密，造成严重损失。

  • arp攻击和arp欺骗实现提前及实现基础

    • 均是通过arp应答来实现的
    • 缺陷——无法对应答的主机提前进行合法身份验证

  • 防御措施

    • 在公司主机上部署安装ARP防火墙软件。
    • 在公司的主机和网关路由器上均进行IP-MAC地址的静态绑定。
