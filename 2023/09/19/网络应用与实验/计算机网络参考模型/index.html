计算机网络参考模型

一、网络分类

  1. 从地域的范围上划分
    • 局域网
    • 广域网
  2. 从用户使用角度上划分
    • 有线网络
    • 无线网络
  3. 从传输速率上划分
    • 普通网络
    • 高速网络
  4. 从企业业务划分
    • 内部网络——私有网络
    • 外部网络——公有网络（internet）

二、分层思想

  1. 网络划分使用分层思想目的：
    • 方便定位问题，进而高效解决问题
    • 降低分析问题的难度，便于维护管理网络

三、如何对网络进行分层——OSI七层参考模型（下一层是为 上一层提供服务的）

  1. 物理层（第一层，也称为最底层）
    • 用于建立、断开、维护物理连接
    • 以比特流的方式进行数据传输，传输信号使用0和1表示
    • 可以数据双向传输（收发数据可以同时进行，即数据的双向传输）
    • 对应的设备——计算机、网卡、网线
  2. 数据链路层（第二层）
    • 把传输的数据帧进行封装和解封装，数据链路层传输的数据称为数据帧
    • 数据帧的地址——源MAC地址、目标MAC地址
    • 数据校验的功能
    • 对应的设备——交换机
  3. 网络层（第三层）
    • 把传输的数据包进行封装和解封装，网络层传输的数据为数据包
    • 数据包的地址——源IP地址、目标IP地址
    • 根据包头字段中的地址进行选路（路由器中的路由表进行数据包的转发）
    • 对应设备——路由器
  4. 传输层（第四层）
    • 实现用户进程间通信（如：在计算机上安装QQ进程）
    • 承上启下（乘上——对应的会话层或应用，启下——对应的网络层）
    • 对应的设备——防火墙
  5. 会话层（第五层）
    • 建立用户间的会话关系
  6. 表示层（第六层）
    • 定义传输数据的标准格式（语法和定义）
    • 对传输的数据进行压缩解压缩、加密解密、编码解码
  7. 应用层（第七层，也称为最高层）
    • 提供与用户的接口（定义两端使用统一程序接口）

四、OSI七层参考模型与TCP/IP模型的关系

  1. OSI七层参考模型，是一种理论参考模型，也成为“经典模型”。
  2. 实际使用中，采用是TCP/IP五层模型，即把会话层、表示层、应用层合并为一层“应用层”。
  3. OSI七层参考模型是由TCP/IP五层模型演进而来的。

五、TCP/IP每层对应的协议

  1. 物理层
    • 底层网络定义的协议均支持
  2. 数据链路层
    • 标准网络定义的协议均支持
  3. 网络层
    • IP（ICMP/IGMP/ARP/RARP）
  4. 传输层
    • TCP UDP
  5. 应用层
    • http/https/smtp/dns/ftp/ssh/tftp

六、数据封装与解封装过程（主机A向主机B发送数据）

发送方（主机A）向接收方（主机B）发送数据前，要进行封装过程如下：

  1. 发送方（主机A）对数据进行封装时，从高层往底层进行逐层封装
  2. 从应用层到传输层，在上层数据的基础上封装TCP头部
  3. 从传输层到网络层，在“TCP头部+上层数据”基础上封装IP头 部
  4. 从网络层到数据链路层，在“IP头部+TCP头部+上层数据”基 础上封装MAC头部
  5. 到最后的物理层，转换成比特流在网络中传输。

接收方（主机B）从发送方（主机A）接收数据前，要进行解封装的过程如下：

  1. 接收方（主机B）对数据进行解封装时，从底层到高层逐层进行解封装
  2. 通过物理层网络，传输比特流数据到数据链路层
  3. 到数据链路层后去掉，MAC头部
  4. 到网络层后去掉，IP头部
  5. 到传输层后去掉，TCP头部
  6. 到应用层后，还原上传数据。

七、PDU（协议数据单元）

  1. 名称解释：PDU是定义TCP/IP的五层模型，每层传输数据的名称
    • 物理层传输的数据：比特流
    • 数据链路层传输的数据：数据帧
    • 网络层传输的数据：数据包
    • 传输层传输的数据：数据段

八、TCP/IP五层模型每层对应的硬件设备

  1. 物理层——网卡、传输介质（双绞线，也叫“网线”）
  2. 数据链路层——交换机
  3. 网络层——路由器
  4. 传输层——防火墙
  5. 应用层——计算机软件
