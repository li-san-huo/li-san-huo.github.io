安装及管理程序

一、Linux应用程序基础

名称解释：应用程序，又称为软件，通过安装应用程序，提供一种服务供用户去访问或使用

  • 应用程序与系统命令之间的关系

    角色 系统命令 应用程序
    文件位置 一般在/bin和/sbin目录中，或为Shell内部指令 通常在/usr/bin和/usr/sbin目录中
    主要用途 一般只在字符操作界面中运行 根据实际需要，有些程序可在图形界面中运行
    适用环境 一般只在字符操作界面中运行 根据实际需要，有些程序可在图形界面中运行
    运行格式 一般包括命令字、命令选项和命令参数 通常没有固定的执行格式

  • 经典应用程序的目录结构

    文件类型 保存目录
    普通执行程序文件 /usr/bin
    服务器执行程序文件和管理程序文件 /usr/sbin
    应用程序配置文件 /etc
    日志文件 /var/log
    应用程序参考文档文件 /usr/share/doc
    应用程序手册页文件 /usr/share/man

  • 常见软件包的封装类型

    文件类型 保存目录
    RPM软件包 扩展名为 “.rpm”
    DEB软件包 扩展名为 “.deb”
    源代码软件包 一般为 “.tar.gz”、“.tar.bz2” 等格式的压缩包，包含程序的原始代码
    绿色免安装的软件包 软件包的扩展名不一，，以TarBall格式的居多
    在压缩包内提供已编译好的执行程序文件
    解开压缩包后的文件即可直接使用

二、RPM包管理工具

  • RPM软件包命名格式
    • 软件名称-版本号-发布次数-硬件平台-后缀名.rpm
    • 获取rpm命令的详细帮助信息
      • man
      • rpm
    • rpm命令可以实现几乎所有对rpm软件包的管理功能
  • rpm命令常见的功能
    • 安装、卸载、查询、更新、验证、维护等功能
  • rpm命令使用
    • 查询已安装的RPM软件包
      • rpm -qa查询linux系统中系统已安装的所有rpm软件包
      • rpm -qa |wc-l 统计出linux系统中已安装的所有rpm包的总数
      • rpm -ql openssh查看openssh软件包在当前系统中安装的所有目录和文件
      • rpm -qi openssh查看openssh软件包的详细信息，包括名称、版本、发布次数等
      • rpm -qc openssh查看openssh软件包在当前系统中安装的配置文件
      • rpm -qf /usr/bin/pwd查看pwd这个命令是由哪个软件包安装的
      • rpm -qd openssh 查看openssh软件包在当前系统中安装的帮助文档
    • 查询未安装的RPM包
      • rpm –qp lynx-2.8.8-0.3.dev15.el7.x86_64.rpm （查看未安装的软件包）
      • rpm –qpi lynx-2.8.8-0.3.dev15.el7.x86_64.rpm （查看未安装软件包的详细信息）
      • rpm –qpl lynx-2.8.8-0.3.dev15.el7.x86_64.rpm （预先查看这个包安装将要生成哪些目录和文件）
      • rpm –qpc lynx-2.8.8-0.3.dev15.el7.x86_64.rpm （预先查看这个包安装将要生成的配置文件）
      • pm –qpd lynx-2.8.8-0.3.dev15.el7.x86_64.rpm （预先查看这个包安装将要生成的帮助文档
  • 安装RPM软件包
    • 命令格式
      • rpm 选项 软件包名称
    • 安装选项
      • -i在当前系统中，安装一个新的软件包
      • -U检查并升级系统中的某个软件包，若该软件包未安装，则安装这个软件包，这时就相当于“-i”这个选项
      • -F检查并更新系统中的某个软件包，若该软件未安装，则放弃更新升级安装操作。
      • -e卸载已安装的软件包
      • -v显示软件包安装过程中的详细信息
      • -h在安装或升级软件包的过程中，以“#”的方式显示安装或升级的进度
      • –force强制安装某个软件包，如安装一个比当前版本更低的一个版本时，会使用强制安装
      • –nodeps可用于安装、升级、卸载一个软件包时，忽略软件包的依赖关系
    • 举例
      • rpm -ivh lynx-2.8.8-0.3.dev15.el7.x86_64.rpm (安装全新的lynx软件包)
      • rpm -e lynx-2.8.8-0.3.dev15.el7.x86_64.rpm （卸载lynx软件包）
      • rpm -q lynx （查看lynx软件包是否卸载）
      • rpm -qpi elinks-0.12-0.37.pre6.el7.x86_64.rpm –nodeps （安装新的软件包，且忽略依赖关系）
  • 维护RPM数据库
    • 重建RPM数据库
      • rpm –rebuilddb（执行效率低）
      • rpm –initdb（执行效率高）
    • 导入验证公钥
      • root@localhost~]# rpm –import/media/cdrom/RPM-GPG-KEY-CentOS-7
      • 作用：导入公钥文件后，安装软件包时，不会提示警告信息，确保安装到系统的RPM包是安全的。
    • 解决软件包依赖关系方法
      • 安装有依赖关系的多软件包，被依赖的软件先安装，可以同时制定多个rpm包进行安装
      • 卸载有依赖关系的多个软件包时，被依赖的软件先卸载，可以同时制定多个rpm包进行卸载
      • 安装或卸载有依赖关系的软件包，不考虑其依赖关系进行安装或卸载使用“–nodeps”选项，忽略依赖关系，进行强制安装或卸载有依赖关系的软件包

三、源代码包编译安装程序

  • 源代码编译安装概述
    • 使用源代码包安装软件的优点
      • 获取较新软件版本，及时修改BUG
      • 根据用户需求，灵活定制软件功能模块
    • 源代码使用场景
      • 企业要求安装较新版本的软件包时
      • 企业业务需求无法及时满足时
      • 应用程序需要添加新的功能时
    • 源代码包格式
      • 源代码包，又称为“Tarball”
      • 格式 tar.gz。tar.bz2
    • 校验源代码完整性的校验工具
      • md5sum https-2.4.50.tar.gz
    • 确认源代码包编译环境
      • gcc编译工具
      • 验证gcc编译工具是否安装 gcc -version
      • 验证make编译工具是否安装 make –version
      • 提前安装apache的依赖包 yum install -y gcc gcc-c++ make apr apr-devel apr-until apr-until-devel pcre pcre-devel
    • 源代码包编译安装的过程
      • 下载源代码包并验证源代码包的安装性 md5sum httpd-2.4.50.tar.gz
      • 解包，使用tar tar zxvf httpd-2.4.50.tar.gz -C /usr/src
      • 配置，使用./configure脚本文件进行配置apache安装路径 cd /usr/src/httpd-2.4.50 ./configure –prefix=/usr/local/apache
      • 编译，将源代码编译并生成二进制文件 make
      • 安装，将生成的二进制文件复制到系统对应的目录下 make install
      • 测试，应用及维护软件
        • 修改apache配置文件/usr/local/apache/conf/ httpd.conf（194行的#去掉）
        • 启动apache服务命令/usr/local/apache/bin/apachectl start
        • 验证apache服务是否启动 netstat -anpt |grep httpd
        • yum insatll -y lynx elinks
        • lynx 或lynx 127.0.0.1
        • elinks 或elinks 127.0.0.1
