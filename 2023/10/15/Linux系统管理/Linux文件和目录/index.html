目录和文件管理

一、目录结构

Linux操作系统中的目录和文件数据被组织为一个树形目录结构，所有分区、目录、文件等都具有一个相同的位置起点—-根目录

  • 根目录

    • 所有分区、目录、文件等的位置起点
    • 整个树形目录结构中，使用独立的一个“/”表示

  • 常见的子目录及其作用

    目录 作用
    /boot 此目录是系统内核存放的目录，同时也是系统启动时所需文件的存放目录
    /bin 这一目录存放了所有用户都可执行的且经常使用的命令，如 cp、ls 等。
    /etc 系统主要的和服务产生的配置文件几乎都放置到这个目录内,例如人员的账号密码文件、各种服务的起始文件等。
    /home 存放所有普通系统用户的默认工作文件夹（即宿主目录、家目录）
    /root 该目录是管理员（超级用户）root 的宿主目录，默认情况下只有 root 用户的宿主目录在根目录下而不是在“/home”目录下
    /sbin 存放 Linux 操作系统中最基本的管理命令，一般管理员用户才有权限执行。
    /usr 存放其他的用户应用程序，通常还被划分成很多子目录，用于存放不同类型的应用程序
    /var 存放系统中经常需要变化的一些文件，如系统日志文件、用户邮箱目录等

二、查看及检索文件

cat 显示文件内容

cat 命令本来用于连接（Concatenate）多个文件的内容，但在实际使用中更多地用于查看文件内容。cat 命令是应用最为广泛的文件内容查看命令。使用该命令时，只需要把要查看的文件路径作为参数即可。

  • cat 显示并连接文件的内容，输出文件的所有内容到屏幕
  • 例子：
    • cat /etc/hosts (hosts文件是linux主机中映射的文件)
    • cat /etc/sysconfig/network-scripts/ifcfg-ens33（网卡对应的配置文件）
    • cat /etc/redhat-release （liunx当前系统版本对应的配置文件）

more 和 less 分页查看文件内容

使用 cat 命令可以非常简单地直接显示整个文件的内容，但是当文件中的内容较多时，很可能只能看到最后一部分信息，文件前面的大部分内容都来不及看到。而 more 和 less命令通过采用全屏的方式分页显示文件，便于我们从头到尾仔细地阅读文件内容。

  1. more

    more 命令是较早出现的分页显示命令，表示文件内容还有更多（more）的意思，less命令是较晚出现的分页显示命令，提供了比早期 more 命令更多的一些扩展功能。两个命令的用法基本相同。

    • more查看文件内容，进入全屏模式，使用快捷键如下：
      • Enter键，逐行滚动查看
      • Space键，向下翻屏键
      • 字母b键，向上翻屏键
      • 字母q建，退出全屏模式，返回命令行

  2. less

    less 命令使用方法与 more 命令基本类似，但是比 more 更好的是，less 命令结合管道符号“|”分屏查看执行命令时输出的信息时，既可以向下翻页，也可以向上翻页。而 more 命令只能向下翻页，不能向上翻页。

    • less查看文件内容，进入全屏模式，使用快捷键如下：
      • Enter键，逐行滚动查看
      • Space键，向下翻屏键
      • 字母b键，向上翻屏键
      • 字母q建，退出全屏模式，返回命令行
      • Page up键，向上翻屏键
      • Page down键，向下翻屏键
      • 上下方向键，向上、向下逐行滚动查看
      • 查找键“/”

head 和 tail 查看文件开头或末尾的部分内容

head 和 tail 是一对作用相反的命令，前者用于显示文件开头的一部分内容，后者用于显示文件末尾的一部分内容。可以使用“-n”选项（n 为具体的行数）指定需要显示多少行的内容，若不指定行数，默认只显示十行。

  1. head查看文件开头的内容
    • head /etc/passwd （默认查看文件的前10行内容）
    • head -n 3 /etc/passwd （查看文件的前三行内容）
    • head -3 /etc/passwd （查看文件的前三行内容）
  2. tail命令的用法
    • 查看文件结尾的部分文件
      • tail /etc/passwd （默认查看文件的后10行内容）
      • tail -n 3 /etc/passwd （查看文件的最后三行内容）
      • tail -3 /etc/passwd （查看文件的最后三行内容）
    • 跟踪文件尾部内容追加或更新的信息
      • tail -f /var/log/messages（跟踪linux系统日志文件结尾追加信息）

wc 和 grep 统计和检索文件内容

  1. wc统计文件内容（行数、单词、字节）

    wc 命令用于统计文件内容中包含的行数、单词数、字节数等信息，使用文件名作为参数，可以同时统计多个文件。较常用的选项如下:

    • wc /etc/passwd (统计出/etc/passwd文件的行数、单词总结、字节数)
    • wc -l /etc/passwd（单独统计出文件的行数）
    • wc -w /etc/passwd（单独统计出文件的单词数）（以空格或制表位作为分隔）
    • wc -c /etc/passwd（单独统计出文件的字节数）

  2. grep检索和过滤文件内容

    grep 命令用于在文件中查找并显示包含指定字符串的行，可以直接指定关键字符串作为查找条件，也可以使用复杂的条件表达式（例如,“^word”表示以 word 开头，“word$”表示以 word 结尾，“^$”表示空行）。

    • grep命令常用的选项
      • -i查找时忽略大小写
      • -v翻转查找，输出与查找条件不相符的行
    • grep 命令查找条件设置
      • “”，使用双引号表示查找的字符串
      • “^…..”，表示以什么开头
      • “…..$”，表示以什么结尾
      • “^$”，表示空行
    • 举例
      • grep“root” /etc/passwd（过滤筛选出/etc/passwd文件中，包含root的行）
      • grep–v “root” /etc/passwd（过滤筛选出/etc/passwd文件中，不包含root字符串的行）
      • grep–i“Root” /etc/passwd（过滤筛选出/etc/passwd文件中，包含root字符串的行，不区分大小写）
      • grep“^root” /etc/passwd（过滤筛选出/etc/passwd文件中，以root字符串开头的行）
      • grep“bash$” /etc/passwd（过滤筛选出/etc/passwd文件中，以bash字符串结尾的行）
      • grep“^$” /etc/passwd（过滤筛选出/etc/passwd文件中，包含空行）
      • grep -v “^$” /etc/passwd（过滤筛选出/etc/passwd文件中，除空行以外的所有行）

三、备份及恢复文档

  • linux系统中常用的压缩文件
    • gzip
      • gzip -9 /etc/passwd（使用gzip高压缩比率压缩passwd文件，产生passwd.gz压缩文件，原文件会消失）
      • gzip -d passwd.gz（使用gzip -d选项解压passwd.gz文件，还原原文件）
      • gunzip passwd.gz（使用gunzip 解压passwd.gz文件）
    • bzip2
      • bzip -9 /etc/passwd（使用gzip高压缩比率压缩passwd文件，产生passwd.bz2压缩文件，原文件会消失）
      • bzip -d passwd.bz2（使用bzip -d选项解压passwd.bz2文件，还原原文件）
      • bunzip passwd.bz2（使用bunzip -d选项解压passwd.bz2文件）
    • gzip与bzip2压缩工具比较
      • gzip压缩比例更高，即压缩出的文件更小
      • 两种压缩工具只能对文件进行压缩，不能对目录进行压缩
      • 实际使用中，常使用gzip压缩工具
  • tar命令（可以对文件和目录进行归档压缩和归档解压缩）
    • 制作归档文件—–制作压缩文件
    • 释放归档文件—-解压已归档的压缩文件
    • tar命令的选项
      • -c创建以.tar结尾的压缩文件
      • -C解压时指定释放的目标位置（已存在目录）
      • -f表示使用归档文件
      • -j调用bzip2程序进行压缩或解压
      • -z调用gzip程序进行压缩或解压
      • -p打包时保留文件及目录的权限
      • -P打包时保留文件及目录的绝对路径
      • -t列表查看包内的文件
      • -v输出解压或压缩过程中的详细信息
      • -x解开.tar的压缩包
    • 举例
      • 制作归档文件（归档压缩tar格式的包文件）进行备份操作
        • tar zvcf passwd.tar.gz /etc/passwd
        • tar jvcf passwd.tar.bz2 /etc/passwd
      • 释放归档文件（解压已打包tar格式的包文件）进行还原操作
        • tar zvxf passwd.tar.gz
        • tar jvxf passwd.tar.bz2
      • 在不解压的情况下，查看归压缩.tar格式包文件的内容
        • tar -tvf passwd.tar.gz
        • tar -tvf passwd.tar.bz2
      • 同时对多个目录进行归档压缩
        • tar zvcf passwd.tar.gz /etc/ /boot
        • tar jvcf passwd.tar.gz /etc/ /boot

四、VI文本编辑器使用

  • vim是vi文本编辑器的增强版本，其也简称为vi编辑器

  • vim工作模式

    • 命令模式
    • 未行模式
    • 输入模式，也称为编辑模式

      End键或”$“键      		 跳转至行首
      跳转到行尾
      行间快速跳转 1G或者gg
