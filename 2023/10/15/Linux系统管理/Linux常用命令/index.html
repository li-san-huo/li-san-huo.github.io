linux常用命令

一、linux命令基础

  • shell程序

    • 这是linux系统中运行的一种特殊程序，在用户与内核之间充当翻译官，当用户登陆系统时会自动加载一个shell程序。
    • linux系统默认使用shell程序对应的文件时 /bin/bash

  • 内部命令与外部命令的区别

    区分类型 内部命令 外部命令
    定义 集成于Shell解释器程序内部的一些特殊指令，也称为内建（Buit-in）指令 在linux系统中，安装其他应用程序所生产的脚本文件或二进制文件
    组成 属于shell解释器的一部分 属于shell解释器之外的部分
    系统文件 没有单独对应的文件系统 每个外部命令在系统中都有一个对应的命令文件
    自动加载 系统重启会自动加载如内存，直接使用 必须找到对应的外部文件所在的位置，再加载到shell中

  • linux系统通用命令格式

    • 命令字 选项 格式

    • 严格区分大小写，即对大小写是敏感的

    • 选项和参数是可以互换位置的

    • 选项是多个，针对多个短格式选项可以合并使用

    • 命令字后可以长格式+短格式混合使用

    • linux命令行的辅助操作

      按键 作用
      Tab 自动补齐
      \ 强制换行
      Ctrl+U 清空至行首
      Ctrl+K 清空至行尾
      Ctrl+L 清屏
      Ctrl+C 取消本次命令编辑
      Ctrl+a 快速将光标定位到行首

二、linux命令帮助

  • help（内部命令）
  • –help（外部命令）
  • man（可用于内部和外部命令）

三、目录与文件的基本操作

命令 作用
pwd 查看系统当前用户工作目录所在的位置
cd 切换用户当前的工作目录
cd ~ 切换到当前用户的宿主目录或家目录
cd 目标位置 系统已存在目录
cd - 返回到上次执行cd命令之前所在的目录

  • 相对路径与绝对路径的区别

    • 相对路径：

      表现形式 说明 示例
      直接使用目录名或文件名 表示当前工作目录中的子目录、文件的位置 grub.conf
      以一个点号”.”开头 表示以当前的工作目录作为起点 ./grub.conf
      以两个点号”..”开头 表示以当前目录的上一级目录（父目录）作为起点 ../vmlinuz
      以”~用户名”形式开头 表示以指定用户的宿主目录作为起点 ~teacher

    • 绝对路径：

      • 以根目录”/“作为起点，进行文件的查询或创建
      • 以绝对路径表示文件路径时，比较繁琐，不够灵活
      • 比较稳定，指定路径比较精确，但不方便迁移

四、ls以列表方式显示目录内容，包括文件和子目录相关的属性信息

  • 使用的参数可以是目录名，也可以是文件名，允许在同一条命令中同时使用多个参数。ls命令选项种类非常多，这里只列出一些最常用的以供参考

    命令 作用
    ls 显示当前目录中包含的子目录、文件列表信息（不包括隐藏目录、文件）
    ls -l 以长格式（Long）显示文件和目录的列表，包括权限、大小、最后更新时间等
    ls -a 显示所有（All）子目录和文件的信息，包括名称以点号“.”开头的隐藏目录和隐藏文件
    ls -A 与-a 选项的作用基本类似，但有两个特殊隐藏目录不会显示，即表示当前目录的“.”和表示父目录的“..”
    ls -d 显示目录（Directory）本身的属性，而不是显示目录中的内容
    ls -h 以更人性化（Human）的方式显示目录或文件的大小，默认的大小单位为字节（B），使用-h 选项后将显示为 KB、MB 等单位。此选项需要结合-l 选项一起使用
    ls -R 以递归（Recursive）的方式显示指定目录及其子目录中的所有内容

  • ls常用组合键

    命令 作用
    ls -hl 以长格式、人性化的方式查看目录中文件及子目录的属性的详细信息
    ls -ahl 以长格式、人性化的方式查看目录中文件及子目录的属性的详细信息，包含隐藏的文件和目录
    ls -Ahl 以长格式、人性化的方式查看目录中文件及子目录的属性的详细信息，包含隐藏的文件
    ls -hl –color 以长格式、人性化的方式查看目录中文件及子目录的属性的详细信息，显示属性信息时显示不同的 颜色
    ls -hl /etc/ns*.conf 以长格式、人性化的方式查询/etc目录中，以ns开头任意多个字符且以.conf结尾的文件
    ls -hl /etc/vi?? 以长格式、人性化的方式查询/etc目录中，以文件名vi开头任意两个字符的文件
    ls -hl /etc/vi??? 以长格式、人性化的方式查询/etc目录中，以文件名vi开头任意三个字符的文件
    • “*”匹配任意的多个字符，”?”匹配任意一个字符

五、alias设置别名

  • alias myls=’ls -alh’
  • 命令myls相当于执行ls -alh命令。别名设置的作用：对经常使用的命令做别名设置，提高命令输入的速度和效率
  • 以命令的方式设置别名，是临时的，只在当前的字符终端生效，其他字符终端无法识别别名

六、du统计目录和文件占用磁盘空间的情况

  • du -sh /boot (统计/boot目录占用磁盘空间的总大小，不显示/boot目录中文件和子文件占用磁盘空间的大小)
  • du -ah /boot（只统计/boot目录占用磁盘空间的总大小，显示/boot目录中文件和子文件占用磁盘空间的大小）

七、mkdir创建空目录

  • mkdir /kgc(在/目录下创建目录kgc)
  • mkdir –p /kgc/kc32/a1/a2（在kgc下创建kc32，在kc32中创建目录a1,在a1下创建目录a2）
  • mkdir –p /beida/kgc/{a/{a1,a2},b/{b1,b2},c/{c1,c2}} （递归创建/beida/目录创建的目录kgc，在kgc目录下创建了目录 a、b、c这三个目录，而目录a下创建了a1和a2，在目录b下创建b1、b2,在目录c下创建c1、c2）

八、touch创建空文件

  • touch命令的作用

    • 更新文件时间标记（针对已存在的文件）
    • 创建空文件（一次可以创建多个）
    格式 作用
    touch a.txt 创建a.txt文件
    touch a.txt b.txt 创建a.txt和b.txt两个文件
    两次touch a.txt 更改a.txt文件的时间标记

九、In创建链接文件

  • 链接文件分类

    • 软连接文件，也称为“符号链接文件”
    • 硬链接文件

  • 创建链接文件的命令格式

    • In -s 源文件 目录位置（软链接）
    • In 源文件 目录位置（硬链接）

  • 例子

    • 软连接的创建：
      • In -s /bin/mkdir /home
      • In -s /bin/ /home
    • 硬链接的创建
      • In /bin/mkdir /home

  • 软链接和硬链接的区别

    类别 软链接文件 硬链接文件
    语法格式 ln -s /bin/mkdir /home
    ln -s /bin/ /home    		 ln /bin/mkdir /home
    创建对象要求 可基于文件创建软链接文件 也可基于目录创建软链接文件 只能基于文件创建硬链接文件
    文件系统要求 源文件和创建的软链接文件可在 同一个文件系统，也可不在同一 个文件系统 源文件和创建的硬链接文件必须 在同一个文件系统内
    删除源文件产生后果 删除源文件后，创建的软链接文 件不可正常使用 删除源文件后，创建的硬链接文 件可正常使用

十、cp复制文件或目录

  • cp 命令用于复制文件或目录，将需要复制的文件或目录（源）重建一份并保存为新的文件或目录（可保存到其他目录中）

  • cp 命令基本格式

    • cp 源文件或目录 目标文件或目录
    命令选项 作用
    cp -f 覆盖目标同名文件或目录时不进行提醒，直接强制复制
    cp -i 覆盖目标同名文件或目录时提醒用户确认
    cp -p 复制时保持源文件的权限、属主及时间标记等属性不变
    cp -r 复制目录时必须使用此选项，表示递归复制所有文件及子目录

  • 例子：

    • cp -i a.txt /kgc：复制的目标位置中与源文件重名的文件进行提示是否覆盖，其是默认选项
    • cp -p a.txt /kgc：复制时保留源文件到目标位置的属性信息不变
    • cp -rf /kgc/kc32 /home：复制到目标位置时强制进行覆盖操作，而不进行提示

十一、rm删除目录或文件

  • rm 命令用于删除指定的文件或目录，在 Linux 命令行界面中，删除的文件是难以恢复的，因此使用 rm 命令删除文件时需要格外小心。rm 命令使用要删除的文件或目录名作为参数。
  • 例子：
    • rm a.txt（提示是否删除a.txt）
    • rm -i a.txt（提示是否删除a.txt）
    • rm -rf /kgc/kc32（递归强制删除kc32目录，不进行提示）
    • rm -r /kgc（递归强制删除kgc目录，进行提示）

十二、mv移动文件或目录

  • mv 命令用于将指定的文件或目录转移位置，如果目标位置与源位置相同，则效果相当于为文件或目录改名。mv命令的基本使用格式如下：
    • mv 源文件或目录 目标文件或目录
      • mv a.txt /kgc
      • mv /kgc/ /home
      • mv a.txt aa.txt

十三、which查找命令或文件存放目录

  • which 命令用于查找 Linux 命令程序并显示所在的具体位置，其搜索范围主要由用户的环境变量 PATH 决定（可以执行“echo $PATH”命令查看），这个范围也是 Linux 操作系统在执行命令或程序时的默认搜索路径。
  • 环境变量的定义：
    • 系统当前用户所拥有的工作环境，包括用户所有能执行的各种命令文件或脚本文件，即定义用户在系统中所拥有的权限
      • which cd
      • which -a ls cd pwd

十四、find命令查找文件或目录

  • find根据不同查找类型（查找表达式）递归进行文件的精细化查找。功能更强大

    查找类型 关键字 说明
    按名称查找 -name 根据目标文件的名称进行查找，允许使用 ”*“ 及 ”?” 通配符
    按文件大小查找 -size 根据目标文件的大小进行查找
    一般使用 “+”、“-”号设置超过或小于指定的大小作为查找条件
    常用的容量单位包括kB（注意k是小写）、MB、GB
    按文件属主查找 -user 根据文件是否属于目标用户进行查找
    按文件类型查找 -type 根据文件的类型进行查找
    文件类型包括普通文件（f）、目录（d）、块设备文件（b）、字符设备文件（c）等

  • 举例：

    • find /etc -name *.conf
    • find /etc -size +1024k
    • find /etc -user root
    • find /etc -type f

  • 使用find命令实现多个条件的查找，需要使用逻辑运算符

    • 逻辑与（查找时条件要同时满足），表示方法“-a”
    • 逻辑或（查找时条件只有满足其一即可），表示方法”-o”
