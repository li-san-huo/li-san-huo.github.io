用户账户和权限管理

一、管理用户账号和组账号

  • linux系统是基于用户身份资源访问进行控制

用户账户

  • 超级用户，root用户账号，在系统中拥有最高权限，是linux系统的超级管理员
  • 普通用户，由管理员用户创建的，拥有一定的权限对系统资源进行访问控制
  • 程序用户，确保系统对应程序的正常运行，该类用户账号不允许登录系统

组账号

  • 基本组（私有组）
    • 定义，创建用户账号的同时，默认也会创建该用户账号对应的基本组（私有组），基本组的名字和 用户账号名字是一样的。
    • 每个用户账号有且只有一个基本组
    • 附加组（公共组）
      • 定义，一个用户账号除了拥有一个基本组外，还可以加入其他组，加入的其他组对于该用户账号来 说，就称为附加组或公共组
      • 每个用户账号可以有多个附加组

用户标识号和组标识号

  • UID（User IDentity，用户标识号)用于唯一区分每一个用户账号
    • 基于centos7系统，创建普通用户默认分配的uid范围从1000~60000
  • GID（Group IDentify，组标识号)用于唯一区分每一个组账号
    • 基于centos7系统，创建组账号默认分配的uid范围从1000~60000
  • root用户账号和组账号，uid和gid始终为0

查询账号信息

  • 用户账号对应的配置文件/etc/passwd
  • 用户配置文件中的每一行都表示一个用户账号的属性信息
    • 字段1：用户帐号的名称
    • 字段2：用户密码字串或者密码占位符“x”
    • 字段3：用户帐号的UID号
    • 所属基本组帐号的GID号
    • 用户全名
    • 宿主目录
    • 登录Shell环境
  • 用户账号对应的密码信息配置文件
    • 密码信息对应的配置文件，也叫“影子文件”，也称为shadow文件/etc/shadow
    • shadow文件每一行对应一个用户账号的密码信息
    • 字段1：用户帐号的名称
    • 字段2：使用哈希加密算法生产的加密字符串
    • 字段3：上次修改密码的时间，从1970.1.1算起到最近一次修改密码时间的间隔天数
    • 字段4：密码的最短有效天数，默认值为0，表示不限制
    • 字段5：密码的最长有效天数，默认值为99999，表示不限制
    • 字段6：提前多少天警告用户密码将过期，默认值为7
    • 字段7：在密码过期之后多少天内禁用此用户
    • 字段8：帐号失效时间，默认值0表示用户账号永不失效
    • 字段9：保留字段（目前未使用）
  • 添加、删除、修改用户账号
    • useradd命令：添加用户账号
      • useradd zhangsan （添加用户张三）
      • useradd -u 1006 -s /bin/sh -e 2022-4-30 lisi（创建用户lisi给他指定uid是1006，有效时间是2022-4-30 登录的 shell）
      • useradd -M xiaoming （添加小明这个用户时不创建他的家目录或属主目录）
      • useradd -G zahngsan xiaoming（添加用户小明时指定该用户账号的附加组为张三）
    • passwd命令
      • passwd zhangsan （设置张三的密码）
      • passwd -d zhangsan（清空或删除张三的密码，实现用户免密登录）
      • passwd -l zhangsan（锁定张三用户，使其无法登录系统）
      • passwd -u zhangsan（解锁张三用户，使其正常登录系统）
      • passwd -S zhangsan （查看用户张三锁定解锁的状态）
    • usermod命令
      • usermod -l lisi zhangsan （把张三修改为李四）
      • usermod -L zhangsan（锁定张三用户，使其无法登录系统）
      • usermod -U zhangsan（解锁张三用户，使其正常登录系统）
      • passwd -S zhangsan （查看用户张三锁定解锁的状态）
    • userdel命令
      • userdel zhangsan（删除用户张三）
      • userdel -r zhangsan（删除用户张三并把其家目录一起删除）
  • 添加、删除组账号
    • groupadd命令
      • groupadd kc32（创建组账号kc32）
      • cat /etc/group （查看组账号信息）
    • gpasswd命令
      • gpasswd -a zhangsan kc32（把用户张三添加到组kc32中）
      • gpasswd -d zhangsan kc32（把用户张三组kc32中删除）
      • gpasswd -M zhangsan,lisi,wangwu kc32（把用户张三、李四、王五添加到组kc32中，且会覆盖原有的用户）
    • groupdel命令
      • groupdel kc32（删除组kc32）
  • 查询账户
    • groups命令
      • groups （查看当前登录系统的用户的属组）
      • groups zhangsan（查看张三用户的属组）
    • id命令
      • id （查看当前登录系统用户的身份标识）
      • id zhangsan （查看张三的uid，gid以及组账号）
    • finger命令
      • finger（查询当前登录账号的详细信息）
      • finger zhangsan（查询张三用户的详细信息）
    • w命令
      • w （查询当前登录账号的详细信息，包括负载，如CPU、内存等）
    • who命令
      • who（只显示用户名、终端、登录时间）
    • users命令
      • users（只显示用户名）
    • whoami命令
      • whoami（查询当前是哪个用户登录的）

二、权限管理

  • linux系统中文件和目录的权限分为：访问权和归属权

    • 访问权

      • 读取 r：允许查看文件内容、显示目录列表
      • 写入 w：允许修改文件内容，允许在目录中新建、移动、删除文件或子目录
      • 可执行 x：允许运行程序、切换目录

    • 归属权

      • 定义：就是文件或目录是由哪个用户创建，所有者就是谁
      • 归属权分为属主,拥有该文件或目录的用户帐号（创建文件或目录的用户）、属组,拥有该文件或目录的组帐号（创 建文件或目录的用户的主账号）

    • 权限的格式和含义

权限设置

  • linux系统下普通文件的默认权限是666，普通目录默认的权限是777
  • chmod命令
    • chmod u+x，g+x，o+x passwd （给文件passwd设置权限，给属主执行的权限，给属组设置执行的权限，给其他 用户设置执行的权限）
    • chmod u-x，g-x，o-x passwd （给文件passwd设置权限，把属主的执行权限去掉，给属组设置的执行权限去掉， 给其他用户的执行权限也去掉）
    • chmod a=rwx passwd （给文件passwd的所有用户的属主、属组赋予读写执行的权限）
    • chmod -R 777 /kgc （给目录kgc以及其子目录和文件均赋予777的权限）
  • chown命令
    • chown zhangsan /root/passwd (把/root/passwd文件的属主改为张三)
    • chown :zhangsan /root/passwd(把/root/passwd文件的属组改为张三)
    • chown zhangsan:zhangsan /root/passwd(把/root/passwd文件的属主和属组均改为张三)
    • chown -R zhangsan:zhangsan /kgc (把/kgc目录以及其子目录和文件的属主和属组均改为张三)
  • umask权限掩码
    • 作用：控制新建的文件或者目录的权限、默认权限去除umask的权限为新建的文件或者目录的权限
    • 权限掩码的默认值：root用户的umask=022、普通用户的umask=002
    • 普通用户创建的文件为644=linux系统中文件的默认权限是666-普通用户的umask是022
    • 普通用户创建的目录为755=linux系统中目录的默认权限是777-普通用户的umask是022
