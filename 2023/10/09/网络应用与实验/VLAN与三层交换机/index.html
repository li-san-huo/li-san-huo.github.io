VLAN与三层交换机

一、VLAN概念

  • 定义：是一种虚拟局域网，代表一种网络地址或网段（即可是公网，即Internet;也可以私网，即内网）。
  • vlan作用
    • 分割广播域
      • 物理分割——通过多台交换机实现对广播域的分割，成本较高，不适用于生产环境。
      • 逻辑分割——通过在交换机上配置VLAN实现对广播域的分割，比较灵活，成本低，使用与生产环境。

二、VLAN优势

  1. 控制广播——PC只能在各自的vlan中接收广播，不能跨vlan接收广播
  2. 提高网络安全性——默认情况下，各自vlan中的PC不能实现互相通信
  3. 简化网络管理——对于管理员配置方便，提高网络管理工作效率。

三、VLAN种类

  1. 静态vlan——基于交换机的接口或端口划分的vlan，称为静态vlan
  2. 动态vlan——基于mac地址划分的vlan，称为动态vlan，不常用，配置成本太高。

四、静态VLAN配置

  1. cisco 2960交换机的vlan范围

    VLAN ID 范围 范围 用途
    0，4095 保留 仅限系统使用
    用户不能查看和使用这些VLAN
    1 正常 Cisco默认VLAN
    用户能够使用该VLAN，但不能删除它
    2-1001 正常 用于以太网的VLAN
    用户可以创建、使用和删除这些VLAN
    1002-1005 正常 用于FDDI和令牌环的Cisco默认VLAN
    1006-1024 保留 仅限系统使用
    用户不能查看和使用这些VLAN
    1025-4094 扩展 仅用于以太网VLAN

  2. vlan范围总结

    • cisco系列交换机vlan范围：0~4095
    • 保留使用的vlan范围：0，4095（两个vlan）；10061024（不能查、不能使用、更不能删）
    • 默认vlan范围，vlan1；vlan1002~vlan1005（默认vlan可以使用vlan，但不能删除）
    • 正常使用的vlan范围：vlan2 ~ vlan1001（增删改查）
    • 扩展使用的vlan范围：vlan1049 ~ vlan4094（增删改查）

  3. 静态vlan创建

    • 创建方法：
      • 数据库配置模式（古老的配置方式）
      • 全局配置模式（通用配置方式）
    • 配置步骤：
      1. 创建vlan
        • vlan database（特权模式）
        • vlan 10 name mingzi
        • exit
        • valn 20（全局配置模式）
        • name mingzi
        • exit
        • no vlan vlan-id（在对应的模式下，不用加vlan的名称）
      2. 将交换机接口加入对应的vlan
        • interface f0/1（全局模式）
        • switchport mode access （定义链路类型为接入链路，也称为access链路）
        • switchport access vlan 10（将交换机接口或端口f0/1加入vlan10）
        • no switchport access vlan 10（进入对应接口模式f0/1）
      3. 验证vlan配置
        • show vlan brief（查看所有vlan配置的概要信息）
        • show vlan id vlan-id（单独查看某个vlan的配置信息）

五、Trunk介绍与配置

  • 在交换网络中，链路的类型共分为两类

    • 接入链路，也称为Access链路
      • 应用场景：PC与交换机相连接使用的链路
    • 中继链路，也称为trunk链路
      • 应用场景：交换机与交换机相连或交换机与路由器相连接的链路

  • trunk的作用：实现相同vlan跨交换机进行通信。

  • vlan跨交换机通信的过程

    • 通过交换机给往其他交换机的数据帧打上VLAN标识，用于区分传输不同VLAN的数据帧，最终实现跨交换机相同vlan下不同PC的相同通信。

    • VLAN标识两种封装协议类型

      • ISL（Cisco的私有标准协议）

        应用场景：仅用于cisco系列的交换机产品，非cisco系列交换机产品不能使用该私有标准封装协议

      • IEEE 802.1Q（通用的标准协议）

        应用场景：即可是cisco系列的交换机产品，也可以是非cisco系列交换机，是一种通用的标准封装协议。

  • 配置trunk链路的步骤与命令

    • 进入接口模式

      • switch(config)# interface f0/24

    • 选择协议的封装类型

      • switch(config-if)# switchport trunk encapsulation dot1q（在三层交换机3560上需要配置，二层交换2960无需配置默认使用的是dot1q）

    • 将接口配置为trunk模式

      • switch(config-if)# switchport mode trunk

    • 查看接口f0/24配置的模式和状态

      • switch# show interface f0/24 switchport

六、三层交换机

  • 三层交换机的作用——使用三层交换技术实现跨VLAN通信

  • 三层交换 = 二层交换（二层交换机-数据帧）+ 三层转发（路由器转发-数据包）

  • 三层交换机是复合产品（交换机+路由器）

  • 三层交换机转发原理：

    • 传统的MLS2-1

      • 3层转发过程中要重新封装2层
      • 三层交换机上，第三层引擎处理数据流的第一个包，交换ASIC从3层引擎中获悉2层重写信息在硬件中创建一个MLS条目，负责重写和转发数据流中的后续数据包。

    • 基于CEF的MLS

      • CEF（CISCO快速转发）是一种基于拓扑转发的模型，包含两个转发用的信息表，一个是FIB（转发信息可以），另一个是邻接关系表。

    • 三层交换机中的两个表

      • 转发信息库（FIB）——相当于路由表

        FIB：FIB类似于路由表，包含路由表中转发信息的镜像，当网络的拓扑发生变化时，路由表被更新，FIB也随之变化

      • 邻接关系表——相当于MAC地址表

        邻接关系表：在网络中，两个节点之间在数据链路层只有一条，即彼此直接相连，则两个节点为邻接关系，邻接关系表就包含了与该节点直接相连的设备的第二层编址信息。

  • 三层交换机虚接口概述

    • 在三层交换机上配置的VLAN接口为虚接口——配置的虚接口的接口IP地址，即是各VLAN中主机的网关地址
    • 虚接口的引入使得应用更加灵活
    • 最终实现跨VLAN通信

七、三层交换机配置

  1. 在三层交换机启用路由功能

    • Switch(config)# ip routing

  2. 配置虚接口的IP

    • Switch(config)# interface vlan-id
    • Switch(config)# ip address ip_address netmask
    • Switch(config-if)# no shutdown

  3. 将三层交换机f0/1接口配置成路由接口命令

    • Switch(config-if)# no switchport

  4. 在三层交换机查看路由表

    • Switch# show ip route
